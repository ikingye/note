    编程 #

    软件设计原则 #

    • Don’t Repeat Yourself (DRY)
    • Keep It Simple, Stupid (KISS)
    • Program to an interface, not an implementation
    • Command-Query Separation (CQS) – 命令 - 查询分离原则
    • You Ain’t Gonna Need It (YAGNI)
    • Law of Demeter – 迪米特法则
    • 面向对象的 S.O.L.I.D 原则
    • Common Closure Principle（CCP）– 共同封闭原则
    • Common Reuse Principle (CRP) – 共同重用原则
    • Hollywood Principle – 好莱坞原则
    • High Cohesion & Low/Loose coupling & – 高内聚， 低耦合
    • Convention over Configuration（CoC）– 惯例优于配置原则
    • Separation of Concerns (SoC) – 关注点分离
    • Design by Contract (DbC) – 契约式设计

    k3s #

    k3d + k3s = k8s

    rancher/k3s

    #

    rancher/k3d

    #

    k3d 则是 k3s 社区创建的一个小工具，可以在一个 docker 进程中运行整个 k3s 集群， 相比直接使用 k3s 运行在本地，更好管理和部署。

    # 安装
brew install k3d

k3d create --server-arg --no-deploy --server-arg traefik -n k3s

export KUBECONFIG="$(k3d get-kubeconfig --name='k3s')"

kubectl cluster-info
    sunny0826/kubecm

    #

    kubecm 帮助你管理杂乱无章的 kubeconfig

    参考：

    • k3d + k3s = k8s perfect match for dev and testing
    • k3d+k3s+kubecm 本地快速搭建与管理集群

    Kubernetes #

    Deployment not ready # 

    kubectl apply -f samples/bookinfo/platform/kube/bookinfo.yaml

    deployment 创建了，not ready。 pod 没有 创建。

    问题： 应该是 kubeconfig 没有配置

    解决方法：

    # 配置 KUBECONFIG
export KUBECONFIG="$(k3d get-kubeconfig --name='k3s1')"

# 删除重新创建
kubectl delete -f samples/bookinfo/platform/kube/bookinfo.yaml
kubectl apply -f samples/bookinfo/platform/kube/bookinfo.yaml
