    HTTP/2 #

    握手方式 #
    h2 #
    • 必须使用 TLS 加密
    • 在 TLS 握手期间会顺带完成 HTTPS/2 协议的协商
    • 如果协商失败（比如客户端不支持或者服务端不支持），则会使用 HTTPS/1 继续后续通讯
    h2c #
    • 不使用 TLS
    • 多了一次基于 HTTP 协议的握手往返来完成向 HTTP/2 协议的升级
    • 一般不建议使用
